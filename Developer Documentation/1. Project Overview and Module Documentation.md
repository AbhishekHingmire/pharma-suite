## 1. Project Overview

### What is Pharma Suite?
Pharma Suite is a comprehensive **Pharmaceutical Distribution Management System** designed for medicine distribution companies. It manages the complete lifecycle of pharmaceutical operations from purchasing medicines from manufacturers to selling to retailers/pharmacies.

### Core Purpose
- **Purchase Management**: Track medicine purchases from pharmaceutical companies with scheme applications
- **Sales Management**: Manage sales to customers with dynamic pricing based on customer types
- **Inventory Control**: Real-time inventory tracking with batch management and expiry monitoring
- **Payment Tracking**: Complete payment lifecycle for both purchases and sales
- **Business Intelligence**: Reports and analytics for business decision making

### Technology Stack
- **Frontend**: React + TypeScript + Vite
- **UI Framework**: Tailwind CSS + shadcn/ui components
- **State Management**: Zustand (for auth), LocalStorage (for data persistence)
- **Routing**: React Router v6
- **Current Data Storage**: Browser LocalStorage (will be replaced with ASP.NET Core APIs)

### 4.3 Purchase Module

**Purpose**: Manage medicine procurement from pharmaceutical companies

**Sub-Modules:**

#### A. New Purchase
**Flow:**
1. **Step 1: Select Company**
   - Display all active companies
   - Show payment terms, contact info
   
2. **Step 2: Add Products**
   - Select product from master list
   - Enter brand name manually (Important: Same generic can have multiple brands)
   - Enter quantity
   - **Scheme Selection (Intelligent)**:
     - System shows available schemes for selected company + product
     - Displays: "Buy X Get Y Free" or "X% Discount"
     - Shows scheme status: Active/Expired/Upcoming
     - Auto-calculates free quantity based on purchase quantity
     - Minimum quantity validation (scheme not applied if qty insufficient)
   - Enter batch number
   - Enter expiry date
   - Enter rate per unit
   - System calculates amount = qty Ã— rate
   
3. **Scheme Application Logic**:
   - Example: Scheme "Buy 10 Get 2 Free"
   - User orders 25 units
   - System calculates: floor(25/10) Ã— 2 = 4 free units
   - Total received: 25 + 4 = 29 units
   - Effective rate: (25 Ã— rate) / 29
   
4. **Payment Details**:
   - **Pending**: Full amount to be paid later (default for staff)
   - **Paid Now**: Requires transaction ID, payment method (Admin only)
   - **Partial Payment**: Enter paid amount, pending auto-calculated (Admin only)
   
5. **Summary & Save**:
   - Subtotal calculation
   - GST @ 12% (configurable)
   - Total amount
   - Save creates:
     - Purchase record
     - Updates inventory (qty + freeQty added)
     - Creates payment record if paid/partial

**Business Rules:**
- Brand name must be entered manually (same medicine, different brands)
- Scheme validation: minimum quantity check
- Free quantity verification required (checkbox for staff to confirm receipt)
- Payment authorization: Only admin can mark as paid
- Inventory auto-updated on purchase save

#### B. Purchase List
**Features:**
- Filter: All / Paid / Pending

**Logic:**

- **Brand Name**: Commercial name by manufacturer (e.g., Dolo-650, Crocin, Calpol)
- **HSN Code**: Harmonized System of Nomenclature (for GST classification)
- **GSTIN**: Goods and Services Tax Identification Number
- **Batch**: Production batch number for traceability
- **FIFO**: First In First Out (oldest stock sold first)
- **FEFO**: First Expiry First Out (nearest expiry sold first)
- **Credit Days**: Days allowed for payment (e.g., Net 30 days)
- **Outstanding**: Pending payment amount
- **Rate Master**: Pricing configuration table
- **Scheme**: Promotional offer from supplier
